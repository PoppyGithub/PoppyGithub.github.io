背景介绍

遇到的坑总结：

1、一定要以你的Github用户名.github.io创建。

假如我没有用PoppyGitHub.github.io而是用了wanglei.github.io，那么当我浏览器访问博客的时候会出现404错误。这里并不是没有部署成功，而是把它部署在了这里:http://PoppyGitHub.github.io/wanglei.github.io。所以，如果想直接PoppyGitHub.github.io访问，那么就需要和用户名保持一致。

2、如果你在执行$ hexo deloy,如果提示 ERROR Deployer not found: git，那说明你没有安装hexo-deployer-git依赖包，不用担心，只需要输入下面命令：

就能解决这个错误了

3、发布的时候一直提示：

Error: Permission denied (publickey). fatal: Could not read from remote repository.

分析这个错误提示大致可以推断出是公钥配置的问题引起的。

重新配置了一下公钥，

执行ssh -T idealife@github.com
也提示：Hi defnngj You’ve successfully authenticated, but GitHub does not provide shell access.
说明配置的公钥没有问题。

执行sudo hexo deploy，提高权限，但是还是报这个错误

后来查了很多文章，发现我生成公钥的命令是ssh-keygen -t rsa -C “idealife@github.com”，没有加sudo,生成的公钥是当前用户的，路径是 /Users/edz/.ssh,而sudo hexo deploy命令执行的时候应该会去读取的root用户的公钥，很显然root下还没有对应的公钥信息生成。

随机执行sudo ssh-keygen -t rsa -C “idealife@github.com”，使用通过sudo cat /var/root/.ssh/id_rsa.pub提取公钥信息并配置到github中，执行sudo hexo d,就部署成功了。
